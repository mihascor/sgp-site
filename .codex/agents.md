# Инструкции для Codex

## Проект

Это сайт-визитка компании ООО «Связьгазпроект» на Next.js.

Основной фокус проекта: аккуратная адаптивная верстка по макету, без сложной бизнес-логики.

## Технологии

- Next.js App Router
- React
- TypeScript
- CSS Modules
- Static assets из `public/`

## Структура

- `app/` — страницы и общий layout
- `sections/` — секции конкретных страниц
- `components/layout/` — общие layout-компоненты: header, footer
- `components/shared/` — переиспользуемые UI-компоненты
- `content/` — markdown-контент
- `public/` — изображения, SVG, PDF и другие статические файлы
- `styles/` — общие стили, если используются

## Правила разработки

- Сохранять существующую архитектуру проекта.
- Не переносить секции между папками без необходимости.
- Для стилей использовать CSS Modules рядом с компонентом.
- Не добавлять новые UI-библиотеки без явной необходимости.
- Не менять глобальные стили, если задачу можно решить локально в module.css.
- Не делать крупные рефакторинги вместе с мелкими правками верстки.
- Не удалять и не переименовывать публичные ассеты без проверки всех ссылок на них.

## Верстка и адаптив

- Проверять изменения на desktop, tablet и mobile.
- Особое внимание уделять ширинам до `480px`.
- Не допускать горизонтального скролла.
- Следить, чтобы текст не прижимался вплотную к краям экрана.
- Использовать существующий компонент `Container`, если секция должна совпадать с общей сеткой сайта.
- Сохранять визуальный стиль проекта: строгий корпоративный дизайн, синий акцент, белые и светло-серые фоны.

## Компоненты

- Header находится в `components/layout/header/`.
- Footer находится в `components/layout/footer/`.
- Глобальные элементы подключены в `app/layout.tsx`:
  - `Header`
  - `Footer`
  - `CookieBanner`
  - `ScrollToTop`

При изменении header/footer учитывать, что они используются на всех страницах.

## Контент

- Текст на сайте преимущественно русский.
- Не менять юридические формулировки, реквизиты, адреса и названия документов без явного запроса.
- Политика обработки персональных данных использует markdown из `content/privacy-policy.md`.

## Проверка

После изменений желательно запускать:

```bash
npm run build
```

Для проверки качества кода можно запускать:

```bash
npm run lint
```

## Ограничения

- В проекте нет API, авторизации и форм.
- Не добавлять серверную логику без необходимости.
- Не добавлять зависимости ради простых CSS/React-задач.
- Не трогать `.next/`, `node_modules/`, логи dev-сервера и сгенерированные файлы.

## Стиль правок

- Делать минимальные точечные изменения.
- Сохранять текущие имена классов, если нет причины их менять.
- Если меняется поведение мобильного меню, проверять закрытие меню при переходах.
- Если меняется футер, учитывать фиксированную кнопку `ScrollToTop`.
- Если меняются размеры или отступы, проверять, что не ломаются соседние секции.

## Команды

Основные команды проекта:

```bash
npm run dev
npm run build
npm run lint
```
